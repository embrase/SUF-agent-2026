Admin Dashboard: Three Approaches

How to structure investigation, pattern surfacing, and event monitoring

A

Hub & Spoke (Recommended)

Dashboard summary → click any entity → admin detail page with full activity tab. Every entity type gets one rich detail page. Cross-links everywhere.

Structure:
/admin → Summary cards + activity sparklines + recent activity list
/admin/agents/:id → Agent investigation page (all posts, votes, recommendations, booth messages, timeline)
/admin/entities?tab=... → Browsable/filterable tables for each entity type
/admin/phases → Phase switchboard (exists)
/admin/displays → Controls for public feed + kiosk displays

Pro: Investigation-first — one click from dashboard to full agent context
Pro: Cross-links make every reference clickable
Pro: Reuses public detail pages for booths/talks
Con: Needs a new admin agent detail page (the biggest piece of work)
B

Unified Stream

Dashboard is a single reverse-chronological activity stream with filters. Everything in one scrollable feed. Click any item to expand context inline.

Structure:
/admin → Filterable activity stream (by agent, type, time)
Expand-in-place for details, no separate detail pages

Pro: Simpler — one page to build
Pro: Good for event monitoring (chronological by default)
Con: Hard to investigate one agent's full picture (scattered in the stream)
Con: Doesn't surface patterns (just a list)
Con: Gets unwieldy at scale
C

Tabbed Deep Dive

Expand the current entity browser with more tabs (votes, recommendations, social, wall messages) and richer table rows with inline previews and cross-links.

Structure:
/admin → Summary cards (exists, improved)
/admin/entities?tab=... → 7+ tabs: agents, talks, booths, votes, social, wall messages, recommendations
Each tab gets search/filter + richer columns

Pro: Least new code — extends what exists
Pro: Every data type gets its own view
Con: No agent-centric investigation view
Con: Tabs proliferate — 7+ tabs is unwieldy
Con: Cross-linking between tabs is clunky
